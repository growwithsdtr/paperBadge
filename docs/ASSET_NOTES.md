# Asset Notes

## Embedded firmware assets

The firmware embeds the generated assets in `src/embedded_assets.h`:

- `badge_en.png`: English static badge fallback.
- `badge_ja.png`: Japanese static badge fallback.
- `profile.png`: optimized profile image fallback.
- `qr.png`: optimized QR fallback.

These arrays are generated by `tools/build_embedded_assets.py` from the current source files in `sample-data/paperbadge/`.

## Current source inputs

The current badge can be regenerated from:

- `sample-data/paperbadge/badge.json`
- `sample-data/paperbadge/profilePhoto.png`
- `sample-data/paperbadge/qr.png`

The asset builder renders the full English/Japanese badge from the JSON, profile image, and QR image. It does not need a pre-rendered 1-2 MB full badge image to build the firmware fallback.

## Reference-only assets

The old `docs/assets/reference/completeBadge.png` full badge render was removed because it was a large visual reference, not a firmware input and not used by PlatformIO builds.

The old ignored generated files below were removed from `generated-assets/embedded/` because they are not referenced by the current builder or firmware:

- `embedded_full_badge.png`
- `embedded_profile.png`
- `embedded_qr.png`

## Size notes

The optimized generated assets are small enough for the current firmware budget:

- `generated-assets/embedded/badge_en.png`: about 38 KB
- `generated-assets/embedded/badge_ja.png`: about 42 KB
- `generated-assets/embedded/profile.png`: about 14 KB
- `generated-assets/embedded/qr.png`: about 4 KB

`src/embedded_assets.h` is larger because it stores the optimized PNG bytes as C arrays.
